AWIPS Shapefile RPM Installation

 

1. Introduction

This document details configuring a Linux system to host the AWIPS Shapefile RPM deployment application.

1.1. Assumptions

  • Target machine has direct access to the Internet.
  • The user performing the installation has an account on VLab and is a member of the VLab ISatSS Community, the VLab ISatSS Redmine Project, and the VLab ISatSS Gerrit Project (in a role that provides at a minimum 'read' access to the git repositories).

1.2. Requirements

  • 64-bit Redhat or CentOS 6 or higher
  • Python (v2.7 minimum)
  • rpm-build
  • subversion
  • git
  • bunzip2, unzip, gcc, gcc-c++
  • If the Shapefile RPM will be generated monthly automatically, cron must be enabled for the target user account

1.3. Preparation

  1. Become the root user ("sudo -iu root", "su -", etc.).
  2. Verify that python 2.7 or greater is installed by type "rpm -q python".
  3. If python is not installed, install it by typing "yum install python".
  4. If python is installed, but the version is older than 2.7, upgrade by typing "yum update python".
  5. Install any missing required packages using the command "yum install <package name>".

 

2. Python Environment Installation

This system makes use of the NWSPY environment hosted by the NWS Python Support Community on VLab.  Follow the instructions for the latest revision (currently p37_n116_2) found in the tab at the top of the community home page (https://vlab.ncep.noaa.gov/group/nws-python-support).  For an installation that will be run from a personal Linux account, the ssh protocol is recommended over https for cloning the git repository.

 

3. ISatSS Installation

The AWIPS Shapefile RPM creator is an ISatSS application requiring installation and configuration of the ISatSS software.  Determine a suitable installation directory.  The installation path you decide will be denoted as ISATSS_HOME in this document.

1. Create the installation directory path:

$ mkdir ISATSS_HOME

$ cd ISATSS_HOME

2. Clone the ISatSS baseline repository.

ISATSS_HOME> git clone ssh://Your.Name@vlab.ncep.noaa.gov:29418/ISatSS

3. Change directory into the root of the repository.  Going forward, this path will be denoded as ISATSS_CODE in this document.

ISATSS_HOME> cd ISatSS

4. Check out the latest branch from the repository (currently p37_n116_v3.0).  Branch names are based on three components: the python version, the numpy version, and the source code version.

ISATSS_CODE> git checkout p37_n116_v3.0

5. Change to the apps sub-directory.

ISATSS_CODE> cd apps

6. Using the editor of your choice, create the file isatss_setup.py and populate it with the following:

# isatss configuration
import os

# path to instance of isatss_support directory.  If not using isatss_support, set to None
conda_home     = None
# path to installed anaconda install.  If not using a local anaconda install, set to None
conda_path     = '<NWSPY>/anaconda'

# path to the python binary to be used by ISatSS
python           = conda_path+'/bin/python'

isatss_home    = '<ISATSS_CODE>'

import_tracebacks = True

add_ons = {'top':'','packages':{},'authid':0,'protocol':'ssh'}
add_ons['top'] = '<NWSPY>/addons'
add_ons['packages']['svn'] = {'stype':'pypi', 'src':'https://files.pythonhosted.org/packages/a1/e7/79a64af11e5d2a1e0996e1e2ce781e297ce711d9835fbe37b43dd6134343/svn-0.3.46.tar.gz'}
 

7. Modify the references to <NWSPY> and <ISATSS_CODE> with the actual full paths for the NWS Python and ISatSS git repo directories as previously defined.

8. Execute the update_rc.py program, which updates the shell .rc file (.bashrc, .cshrc, or .tcshrc) file with an alias for each ISatSS application.

ISATSS_CODE/apps> npy update_rc.py

9. Source the shell .rc file.

source ~/.bashrc (or ~/.cshrc) depending on your default shell

10. Confirm the installation by running the isatss command with no arguments.  A commandline help summary should be displayed.

ISATSS_CODE/apps> cd
